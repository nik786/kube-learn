üëâApplication Load Balancer (ALB): Best for HTTP/HTTPS traffic with advanced routing.
üëâNetwork Load Balancer (NLB): Optimized for TCP/UDP traffic with ultra-low latency.
üëâClassic Load Balancer (CLB): Older generation, ideal for simple use cases.

üö¶2.When would you choose ALB over NLB or CLB?
üí°Use ALB for HTTP/HTTPS traffic when:
‚úîYou need features like path-based or host-based routing.
‚úîYou want integration with services like Lambda or Web Sockets.
üí°Use NLB for:
‚úîHigh-performance, low-latency traffic.
‚úîProtocols like TCP/UDP or when handling millions of requests/sec.

üìÇ3.What is a Target Group in ALB/NLB, and how does it work?
A Target Group is a set of resources (like EC2 instances or containers) that receive traffic from a Load Balancer.
üîçALB: Routes traffic to targets based on listener rules (e.g., path /api).
üîçNLB: Distributes traffic to targets without inspecting packets.

üîÑ4.What is Cross-Zone Load Balancing?
Cross-Zone Load Balancing distributes traffic evenly across all registered targets in all availability zones, regardless of the request origin.
üí°This improves performance and reduces bottlenecks in multi-AZ setups.

üîë5.What is SSL/TLS Termination, and how is it implemented in AWS ELB?
SSL/TLS Termination occurs when the Load Balancer handles encryption/decryption, offloading this task from backend servers.
üõ†Implementation: Upload your SSL certificate to ELB via ACM (AWS Certificate Manager). ELB manages secure traffic on behalf of your application.

Auto Scaling works with ELB to:
‚úîAutomatically add or remove instances based on demand.
‚úîRegister and deregister instances with the Load Balancer dynamically.

üíª7.How does path-based routing work in an Application Load Balancer?
Path-based routing directs traffic to specific Target Groups based on the request URL.
üí°For example:
‚úî/app1 ‚Üí Target Group 1
‚úî/app2 ‚Üí Target Group 2
